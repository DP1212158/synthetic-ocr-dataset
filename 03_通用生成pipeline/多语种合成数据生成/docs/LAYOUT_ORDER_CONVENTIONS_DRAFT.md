# VL11 版面排序约定 + 变体复杂度配比（草案 · 待确认）

> 目的：把"复杂版面顺序有歧义"变成"有唯一规范可依"，并通过**多简单、少复杂**降低出错面。
> 确认后作为 v3 模板编排 `reading_order` 的唯一依据。

## A. 通用排序规则（所有变体）

`reading_order` 编成 `1..N`：
1. 先按 region 顺序：报头/标题 → 正文 → 侧栏/边注 → 页脚。
2. 区域内按 column：逻辑列 0→N（列 0 = 最先读）。方向只改视觉摆放：
   - ltr：列 0 在最左；rtl：列 0 在最右；竖排(蒙古)：列 0 在最左，列内上→下。
   - **同一模板 + 同一套序号**在 6 语种通用，RTL/竖排只镜像，不改序号。
3. 列内按自然先后（上→下）。
4. 图注 caption 紧跟其所属图（`caption_of`）；图本身不编号。
5. 非 OCR 块（image/rule/divider/answer_area/table_structure）不编号。
6. 表格：表头行在前，单元格按行主序；表单按"标签→值"成对。

## B. 复杂版面的规范约定（消歧的核心）

每个"多流"区域在模板里显式标 `flow` 类型，顺序据此确定，杜绝歧义：

- **continuous_columns（连续跨栏正文，如书籍双栏）**：读满第 0 列再第 1 列……列内上→下。
- **stories（多条独立报道/卡片，如报纸/杂志）**：每条报道是一个单元（标题→其正文→其图注读完），单元之间按"栏序→栏内上→下"。即先读完一条，再读下一条，不在标题间跳。
- **main_plus_side（主栏 + 侧栏/边注）**：先读完主栏，再读侧栏/边注（region 顺序保证）。
- **form_grid（表单登记）**：按字段的行主序，成对"标签→值"。
- **table（表格）**：表头行→数据行行主序；表格整体占一段连续序号。
- **exam（考卷）**：题干编号，答题区 answer_area 不编号（非 OCR）。
- **figure_caption**：caption 排在其锚点文本块之后。

歧义收敛原则：**能用 stories/continuous_columns 明确表达的，一律在模板显式标注**；不依赖任何渲染期推断。

## C. 变体复杂度配比（多简单、少复杂）

三档复杂度：
- **S 简单**：单栏 / 线性上→下。顺序天然唯一、几乎零出错。
- **M 中等**：主栏+侧栏，或连续双栏。顺序明确、低风险。
- **C 复杂**：3+ 栏 / 多独立报道 / 密集表单表格。歧义风险最高。

目标：全 72 变体中 **S 占多数、C 尽量少**；即便报纸/杂志/表单这类天生偏复杂，也**优先用 2 栏 + stories 单元序**，避免密集 4 栏。

每类 6 变体的建议配比（S / M / C）：
- 01 报纸页：1 / 3 / 2（去掉密集四栏，最多 2 条 C）
- 02 证书证明：5 / 1 / 0
- 03 考试卷：4 / 2 / 0
- 04 标牌海报：5 / 1 / 0
- 05 书籍页：3 / 3 / 0
- 06 教科书页：3 / 3 / 0
- 07 杂志期刊：1 / 3 / 2
- 08 学术文献：2 / 4 / 0（表格类算 M）
- 09 历史古籍：3 / 3 / 0
- 10 公告通知：5 / 1 / 0
- 11 复杂表单：1 / 2 / 3（表单天生结构化，已尽量压 C）
- 12 手写信件：6 / 0 / 0

合计约 **S 39 / M 26 / C 7**（≈54% / 36% / 10%）。C 只集中在报纸/杂志/表单。

## E. 中文双语混杂机制（VL11 重建，已确认）

> 事实校正：VL10/当前主线**不含中文**（`max_cjk_ratio=0.02`，主动清除），0% 混杂。历史 VL2–VL6 的注入代码已随旧脚本删除。VL11 **新建**该机制。
> 目的：数据用于少数民族 OCR 检测，对抗两类退化——(1) 高资源偏见致模型注意力偏向中文、漏检少数民族文字；(2) RTL/竖排空间顺序错乱。

- **比例**：50% 文档 mixed（含中文）/ 50% pure（纯目标语）。
- **中文占比**：mixed 文档整页 CJK 字符占比落在 **0.10–0.30**（该文档的 `max_cjk_ratio` 相应放开；pure 文档仍 ≤0.02）。
- **粒度 = 块级双语混排**：mixed 文档挑一部分**块**渲染中文（优先标题/栏目/图注 + 部分正文段），其余块保持目标语，形成结构性"少数民族语+中文"双语版面。不使用纯词级随机噪声。
- **标注**：每块按其真实文字体系正常出可 OCR 标注框（中文块与少数民族块都要被检测到）；label 记录 `language_mix_mode`(pure/mixed)、`target_cjk_ratio`、每块的主文字体系。
- **顺序**：中文块与目标语块统一进 A/B 节的显式 `reading_order`，方向无关。
- **适用**：全部 7 语种（含蒙古语；蒙古语中文放横排页眉/图注或竖排列，保守）。
- **中文语料 = 按类手工库（语域贴类）**：`00_共享资源/中文语料/zh_content_bank.json`，按 12 类分别编写语域贴合的中文（标题/栏目/图注/字段/元数据 + 可拼接的正文句）。短/结构块取库中对应角色；长中文正文由 `paragraph_sentences` 随机拼接。**离线、可复现、无外部依赖**。（zh.wikipedia 随机采集经共享代理触发 HTTP 429 且主题不贴类，已弃用。）每类的中文库与该类 v3 模板由同一子 agent 一起产出。
- **QA**：`qa_language_validity --allow-mixed-cjk` 断言 mixed 文档 CJK 落在 0.10–0.30、pure 文档无 CJK。
