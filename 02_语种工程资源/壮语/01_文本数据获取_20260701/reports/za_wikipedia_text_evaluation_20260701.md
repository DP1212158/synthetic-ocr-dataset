# 壮语 Wikipedia 文本源评估

## 来源

- 来源：Zhuang Wikipedia
- API：`https://za.wikipedia.org/w/api.php`
- 输出目录：`data/za_wikipedia_allpages_sample_20260701`
- 文本文件：`records.jsonl`

## 本轮结果

- 扫描标题数：300
- 保留记录数：80
- 拒绝记录数：220
- 总字符数：11726
- 平均长度：146.57
- 中位长度：159.0
- 平均壮文拉丁字符匹配比例：0.7974
- 最低匹配比例：0.6723
- 最高匹配比例：0.8706

## 判断

`za.wikipedia.org` 可以作为第一批合成数据的文本源，但不算充足：

- 优点：文本确实以现代标准壮文拉丁拼写为主，过滤后语言比例较高。
- 问题：单条平均长度偏短，且逐页抓取触发了 HTTP 429 限速。
- 本轮用途：足够支撑 `12类 x 6张` 的 v1 样本生成。
- 后续建议：若扩展到更大规模，应接入第二文本源，并把 MediaWiki 抓取改成批量 pageids/extracts，降低 429 风险。

## 过滤规则

- 现代壮语按拉丁字母脚本处理。
- 保留条件：
  - 文本长度 >= 80
  - 壮语拉丁形态估计比例 >= 0.30
- 本轮不处理方块壮字/古壮字。
